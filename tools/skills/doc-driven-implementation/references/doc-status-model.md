# 文档状态模型

当一个仓库使用文档状态闸门时，使用这份参考说明。

## 有效状态

- `已接受`
  表示设计或决策已经拍板，可以作为实现依据。
- `已生效`
  表示治理规则、目录入口或流程规范已经正式启用。
- `已落地`
  表示代码和文档已经同步完成，反映的是当前实现现实。

## 无效状态

- `草案`
  早期草稿。可以辅助讨论，但不能直接支撑正式实现。
- `评审中`
  正在审阅。可以帮助比较和反馈，但不能直接支撑正式实现。
- `已废弃`
  仅保留历史价值，不能作为当前方案依据。

## 决策规则

改代码前，先明确回答这个问题：

`本次实现是由哪一份有效文档授权的？`

如果回答不出来，就不要继续实现。
