# V1.2.0 设备图片上传设计（管后端分层与规则落地，开始实现前先查）

## 设计结论

- 接口层提供：创建设备、查询设备、上传设备图片三个端点
- 服务层负责设备存在性校验与“最多 3 张图片”规则
- 仓储层使用内存实现，保留 `repository` 抽象，便于说明企业项目分层
- 迁移脚本按 `db/migration` 目录组织，配套在 `docs/sql` 和 `docs/upgrade` 保留事实与升级步骤
- 当前示例只接收图片 URL，不处理真正文件上传

## 关键约束

- 第 4 张图片上传返回 `409 Conflict`
- 查询不存在设备返回 `404 Not Found`
