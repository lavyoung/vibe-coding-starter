## 1. 方案摘要

- 使用 `POST /tasks/{taskId}/attachments` 接口录入附件
- 使用内存存储示例化 `task.attachments`
- 由领域层负责“最多 3 个附件”的规则

## 2. 关键规则

- `filename` 和 `url` 都不能为空
- 单任务附件上限为 3
- 任务不存在时返回 `404`

该设计已经在示例代码中实现，因此状态为 `已落地`。
