# 贡献指南

## 适用范围

这个仓库是一套面向“文档优先 + 人类/AI 协作交付”的通用 starter kit。

好的贡献通常会改进以下内容之一：

- 可复用的项目治理规则
- docs 目录结构
- 文档状态与同步流程
- 新会话 / 实现阶段提示词
- 通用 Codex skills
- 帮助别人落地使用的示例

## 提 PR 前请先确认

1. 说明你要解决什么问题。
2. 明确本次改动属于哪类：
   - starter 结构
   - 文档流程
   - 提示词更新
   - skill 更新
   - 示例项目材料
3. 保持改动可复用，不要把某个项目的私有业务规则直接塞进默认模板。

## 贡献规则

### 1. 通用模式优先于本地习惯

如果一条规则只适用于某个团队、某家公司或某个技术栈变体，就不要把它直接写成默认模板，除非明确标为可选项。

### 2. 不要削弱状态闸门

不要弱化这条核心规则：

只有“有效文档”才能作为正式实现依据。

### 3. 提示词、文档、skill 要保持一致

如果你修改了以下任一内容：

- 状态语义
- 读取顺序
- 文档同步流程
- starter skills

那么同一个 PR 里也要同步更新相关 docs 和 prompts。

### 4. skill 保持通用

starter 自带的 skill 不应硬编码以下内容：

- 某一个仓库名
- 某一个构建工具路径
- 某一家公司的包结构
- 某一台机器的本地路径

## PR 期望内容

请在 PR 里至少写清：

- 改了什么
- 为什么这会提升复用价值
- 哪些文档或 skills 被同步更新了
- 这次改动的权衡点是什么

如果仓库里已经有 PR 模板，请优先按模板填写。
